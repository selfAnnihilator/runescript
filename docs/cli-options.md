# CLI Options

## Command Line Interface

RuneScript provides several command-line options for different use cases.

## Basic Usage
```bash
java -jar runescript.jar [options] [file.rn]
```

## Available Options

### Execute Script
```bash
java -jar runescript.jar script.rn
```
Execute a RuneScript file and run the program.

### Debugging Options

#### Tokenization
```bash
java -jar runescript.jar --emit-tokens script.rn
```
Show how the source code is broken down into tokens.

**Example Output:**
```
IDENTIFIER print null
LEFT_PAREN ( null
STRING "Hello, World!" Hello, World!
RIGHT_PAREN ) null
SEMICOLON ; null
EOF  null
```

#### Abstract Syntax Tree (AST)
```bash
java -jar runescript.jar --emit-ast script.rn
```
Show the structure of the parsed code.

**Example Output:**
```
Expression[expression=Call[callee=Variable[name=IDENTIFIER print null], paren=RIGHT_PAREN ) null, arguments=[Literal[value=Hello, World!]]]]
```

#### Bytecode Generation
```bash
java -jar runescript.jar --emit-bytecode script.rn
```
Show the generated bytecode instructions.

**Example Output:**
```
Bytecode instructions:
0: CONSTANT
1: 0
2: PRINT
3: NIL
4: RETURN
```

### Interactive Mode
```bash
java -jar runescript.jar
```
Start the interactive REPL (Read-Eval-Print Loop).

**Example Session:**
```
RuneScript REPL - Type 'exit' to quit
> print("Hello");
Hello
Returned: null
> let x: int = 5;
> print(x);
5
Returned: null
> exit
```

## Examples

### Running Different Scripts
```bash
# Run a simple hello world
java -jar runescript.jar examples/hello.rn

# Run arithmetic examples
java -jar runescript.jar examples/arithmetic.rn

# Run variable examples
java -jar runescript.jar examples/variables.rn
```

### Debugging a Problematic Script
```bash
# Check tokenization if there are syntax errors
java -jar runescript.jar --emit-tokens buggy_script.rn

# Check AST if there are structural issues
java -jar runescript.jar --emit-ast buggy_script.rn

# Check bytecode if there are runtime issues
java -jar runescript.jar --emit-bytecode buggy_script.rn
```

## Error Handling

### Common Error Messages

#### Syntax Errors
```
[line 5] Error Expected expression.
```
- **Cause**: Invalid syntax at line 5
- **Solution**: Check the syntax around line 5

#### Type Errors
```
Cannot assign ErrorType[] to IntType[]
```
- **Cause**: Type mismatch in assignment
- **Solution**: Check that types match in assignment

#### Runtime Errors
```
Operands must be numbers.
```
- **Cause**: Trying to perform arithmetic on non-numbers
- **Solution**: Ensure operands are numbers

## Tips for Using CLI Options

1. **Start with simple scripts** to understand the basic execution
2. **Use --emit-tokens** when facing syntax errors
3. **Use --emit-ast** to understand how code is parsed
4. **Use --emit-bytecode** to see how code is compiled
5. **Use REPL mode** for quick testing and experimentation

## Performance Considerations

- The JAR file loads quickly for each execution
- Debugging options add overhead but provide valuable insights
- Interactive mode keeps the VM loaded for faster command execution
